Для выполнения работы была выбрана среда контейнеризации **Docker Desktop**. В ходе установки возникли сложности с доступом к официальному репозиторию Docker Hub (ошибки сетевых блокировок), в связи с чем был применен метод **кастомной сборки образа** на базе легковесного дистрибутива **Alpine Linux**.

## 2. Работа в Figma и подбор тем оформления

Проведен анализ макета альтернативной версии сайта кафедры в Figma. На основе визуальных решений (левое навигационное меню, строгий академический стиль) подобраны следующие темы CMS Grav:

*   **Тема №1: Learn2**  
    [Ссылка на тему Learn2](https://getgrav.org)  
    **Комментарий:** Структура темы (фиксированное левое меню с древовидной навигацией) максимально соответствует макету в Figma. Это оптимальный вариант для размещения учебных пособий и методических материалов кафедры.

*   **Тема №2: Quark**  
    [Ссылка на тему Quark](https://getgrav.org)  
    **Комментарий:** Официальная адаптивная тема Grav. Выбрана за строгий дизайн, который наиболее адекватен специфике внешнего оформления сайта вуза или факультета.

## 3. Референсы и проектирование информационной страницы

Для проектирования структуры информационной страницы «О кафедре» в качестве референсов были изучены официальные порталы ведущих вузов:
1. [Университет ИТМО](https://itmo.ru) — пример современной плиточной структуры.
2. [НИУ ВШЭ (Факультет компьютерных наук)](https://hse.ru) — пример удобной навигации по образовательным программам.
3. [МГУ им. Ломоносова](https://msu.ru) — эталон строгого академического портала.

**Результат:** На основе референсов подготовлена концепция страницы «О кафедре». Выбранные темы оформления позволяют эффективно визуализировать контент согласно стандартам высших учебных заведений.

## Вывод
В ходе работы была успешно развернута инфраструктура для CMS Grav в контейнеризированной среде Docker. Проведен сравнительный анализ дизайна Figma и доступных тем оформления, выбраны наиболее подходящие варианты для сайта кафедры.
