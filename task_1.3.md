## Фрагмент руководства системного программиста

### 1. Общие сведения о программе

Программное обеспечение предназначено для функционирования официального веб‑сайта кафедры информационных технологий и электронного обучения (ИТиЭО) РГПУ им. А. И. Герцена, доступного по адресу https://ict.herzen.spb.ru.[web:60][web:66]  

Сайт реализован на основе **Grav CMS** — flat‑file системы управления содержимым, использующей язык PHP и набор библиотек (Twig, YAML, Markdown и др.).[web:79] Программа обеспечивает публикацию информации о кафедре, учебной и научной деятельности, а также предоставление сервисов для студентов и абитуриентов.

---

## 2. Структура программы

*(Раздел 2 руководства системного программиста по ГОСТ 19.503‑79.)*[web:73][web:77][web:81]

### 2.1. Общие сведения о структуре

Структура программного обеспечения построена по трёхуровневой схеме:

- **серверный уровень** – веб‑сервер и интерпретатор PHP, обрабатывающие HTTP‑запросы;[web:80]  
- **уровень ядра CMS Grav** – базовая логика управления контентом, конфигурацией, плагинами и шаблонами;[web:79]  
- **уровень пользовательского контента и представления** – файлы страниц, конфигурации сайта, темы оформления, плагины и статические ресурсы (CSS, JS, изображения).[web:79]

Программа использует файловую архитектуру хранения данных (flat‑file): все страницы, настройки и служебные данные представлены в виде файлов на диске, без реляционной базы данных.[web:79][web:80]

### 2.2. Составные части программы

#### 2.2.1. Веб‑сервер и среда выполнения PHP

Веб‑сервер (Apache, Nginx или встроенный сервер PHP) выполняет следующие функции:[web:80]

- приём и маршрутизация HTTP/HTTPS‑запросов пользователей;  
- запуск интерпретатора PHP для обработки запросов;  
- возврат сгенерированных HTML‑страниц и статических ресурсов в браузер пользователя.

Интерпретатор PHP обеспечивает выполнение сценариев Grav CMS, загрузку библиотек, работу с файловой системой и взаимодействие с внешними сервисами.[web:79]

#### 2.2.2. Ядро CMS Grav

Ядро Grav реализовано на языке PHP и включает следующие подсистемы:[web:79]

- маршрутизация URL‑запросов и определение соответствующей страницы;  
- загрузка и объединение конфигурации из YAML‑файлов (`system/`, `user/config/`);  
- обработка содержимого страниц на основе файлов Markdown и frontmatter;  
- система событий и плагинов, обеспечивающая расширяемость функциональности;  
- интеграция с шаблонизатором Twig для формирования HTML‑представления.

Ядро использует компоненты фреймворка Symfony (контейнер зависимостей, обработка событий, консольные команды).[web:79]

#### 2.2.3. Пользовательская область и контент

Пользовательская область расположена в каталоге `user/` и включает:[web:79]

- `user/pages/` – иерархию страниц (разделы «О кафедре», «Учебная деятельность», «Наука», «Абитуриенту» и др.), каждая страница представлена каталогом с Markdown‑файлом и YAML‑метаданными;[web:66]  
- `user/config/` – конфигурационные файлы сайта, тем и плагинов (формат YAML);  
- `user/accounts/` – учётные записи администраторов сайта;  
- `user/data/` – служебные данные, создаваемые плагинами (результаты форм, кеш и т.п.).

Содержимое страниц формирует информационное обеспечение сайта кафедры: описания кафедры, учебных программ, новостей, научных проектов и т.д.

#### 2.2.4. Темы оформления и система шаблонов

Темы оформления находятся в каталоге `user/themes/` и включают:[web:79][web:17]

- Twig‑шаблоны (`templates/`), определяющие структуру HTML‑страниц;  
- файлы стилей (`css/`), задающие внешний вид элементов;  
- клиентские скрипты (`js/`), реализующие интерактивные элементы интерфейса;  
- графические ресурсы (логотипы, иконки, изображения).

Активная тема определяется в конфигурации и отвечает за визуальное представление разделов сайта кафедры.

#### 2.2.5. Плагины и модули расширения

Плагины размещаются в каталоге `user/plugins/` и расширяют базовую функциональность Grav:[web:79][web:17]

- обработка форм обратной связи;  
- организация поиска по сайту;  
- создание RSS‑лент;  
- поддержка мультиязычности;  
- интеграция с внешними сервисами (почта, аналитика);  
- дополнительные типы страниц и виджеты.

Плагины подключаются к системе событий ядра CMS и могут влиять на процесс обработки запросов и рендеринг страниц.

#### 2.2.6. Статические ресурсы

Статические ресурсы включают:

- файлы CSS для оформления страниц;  
- скрипты JavaScript для работы меню, модальных окон, валидации форм;  
- изображения сотрудников, логотипы, иллюстрации к новостям;  
- шрифты и иконки.

Ресурсы подключаются из Twig‑шаблонов и отдаются браузеру совместно с HTML‑страницами.

### 2.3. Связи между составными частями

Процесс обработки запроса можно описать следующим образом:

1. Пользователь вводит адрес страницы сайта кафедры в браузере.  
2. Веб‑сервер принимает HTTP/HTTPS‑запрос и передаёт его интерпретатору PHP.  
3. Ядро Grav определяет маршрут, находит соответствующую страницу в каталоге `user/pages/` и загружает её содержимое.[web:79][web:80]  
4. Система конфигурации применяет глобальные и локальные настройки из YAML‑файлов.  
5. При необходимости вызываются плагины, подписанные на события обработки запроса.  
6. Данные передаются в Twig‑шаблон, определённый активной темой.  
7. Twig формирует итоговый HTML‑документ, к которому подключаются CSS, JavaScript и графические ресурсы.  
8. Веб‑сервер отправляет сгенерированную страницу пользователю.

### 2.4. Связи с другими программами

Программа взаимодействует со следующими внешними компонентами:

- **Веб‑браузер пользователя** – отображение страниц, выполнение JavaScript‑кода, отправка форм.  
- **Почтовый сервер (SMTP)** – отправка уведомлений с сайта (например, сообщений из форм обратной связи).  
- **Системы аналитики** (при использовании соответствующих скриптов) – сбор статистики посещаемости и поведения пользователей.

### 2.5. Хранение данных

Хранение данных реализовано без применения реляционной СУБД; все данные размещаются в файловой системе.[web:79] Основные типы данных:

| Тип данных                        | Формат   | Каталог                |
|-----------------------------------|----------|------------------------|
| Контент страниц                   | Markdown | `user/pages/`          |
| Метаданные страниц (frontmatter)  | YAML     | в начале Markdown‑файлов |
| Глобальные настройки сайта        | YAML     | `user/config/`         |
| Учётные записи администраторов    | YAML     | `user/accounts/`       |
| Данные плагинов (формы, кэш и др.)| YAML/JSON| `user/data/`           |

Такой подход обеспечивает простое резервное копирование и перенос сайта между серверами.
