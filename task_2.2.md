# Задание 2.2  
## Подборка расширений для CMS Grav и план установки одного из плагинов

### Исходные данные

На основе анализа программного решения сайта кафедры ИТиЭО (https://ict.herzen.spb.ru/) и функций, сформулированных в задании 1.2, было выбрано направление:

> **Публикация тематической справочной информации**  
> (электронный справочник / библиотека материалов для студентов и преподавателей).

Для реализации этого направления на CMS Grav были подобраны плагины, которые помогают:

- структурировать справочные материалы;
- организовать поиск и навигацию по ним;
- управлять данными через админ‑панель.

# Задание 2  
## Установка, настройка и демонстрация работы плагина SimpleSearch

### Выбранное направление

Для детальной демонстрации был выбран плагин **SimpleSearch** в контексте направления:

> **Публикация тематической справочной информации**.

Задача плагина — обеспечить удобный поиск по справочным материалам сайта кафедры (описания дисциплин, инструкции, регламенты, ответы на частые вопросы и т.п.).

### Важное примечание о среде

Попытка развернуть Grav в Docker‑контейнере на локальном компьютере завершилась без ошибок на стороне Docker, однако при обращении к `http://localhost:8080` сервер Apache возвращал сообщение **404 Not Found** (страница не найдена). Это означает, что:

- контейнер был успешно запущен (лог показывает запуск Apache),  
- но DocumentRoot или путь к Grav внутри контейнера был настроен иначе, чем ожидалось (вероятно, Grav установлен не в корне, а в подкаталоге, либо требуется дополнительная настройка Apache).[web:80]

В результате из‑за ограничений среды **практическая проверка работы плагина в браузере не выполнена**, однако ниже приведён подробный алгоритм установки и настройки SimpleSearch, который будет использован при последующей успешной конфигурации локального Grav (например, при запуске без Docker, напрямую через PHP‑сервер).

---

## План установки и настройки SimpleSearch в локальном Grav

### 1. Подготовка локального Grav (вариант без Docker)

Для повторной попытки запуска Grav в более простой конфигурации рекомендуется:

1. С сайта Grav скачать архив **Grav + Admin** (Grav Core + Admin Plugin).[web:79]  
2. Распаковать архив в каталог, например `C:\grav-site`.  
3. Установить PHP и убедиться, что он доступен в переменной PATH.  
4. В командной строке, в каталоге `C:\grav-site`, выполнить:  
   `php -S localhost:8000 -t .`[web:80]  
5. Открыть в браузере адрес `http://localhost:8000/admin` и создать учётную запись администратора.

*(Эти шаги описывают план действий; при фактическом выполнении можно сделать скриншоты запуска PHP‑сервера и первой загрузки админ‑панели.)*

---

### 2. Вход в админ‑панель Grav

1. Открыть в браузере адрес `http://localhost:8000/admin`.  
2. Ввести логин и пароль администратора, созданные при первой настройке.  
3. Перейти в интерфейс управления сайтом.


---

### 3. Установка плагина SimpleSearch

1. В левом меню админ‑панели выбрать пункт **“Plugins”**.  
2. Нажать кнопку **“Add”** или **“Add New”** для перехода к каталогу плагинов.  
3. В строке поиска ввести `SimpleSearch`.  
4. В списке найденных расширений выбрать **SimpleSearch** (официальный плагин).  
5. Нажать кнопку **“Install”** и дождаться завершения установки.

**Скриншот (планируется):**  
– список плагинов с найденным SimpleSearch;  
– сообщение об успешной установке.

---

### 4. Настройка SimpleSearch

1. В списке установленных плагинов найти **SimpleSearch** и нажать кнопку **“Configure”**.  
2. Проверить и при необходимости изменить основные параметры:
   - **Enabled** – плагин включён;  
   - типы страниц, по которым нужно выполнять поиск (например, `default`, `blog` и другие используемые типы шаблонов);  
   - количество результатов, отображаемых на странице выдачи.  
3. Сохранить изменения (кнопка **“Save”**).

**Скриншот (планируется):** окно настроек SimpleSearch с отмеченными параметрами.

---

### 5. Добавление формы поиска на страницу справочника

1. В админ‑панели перейти в раздел **“Pages”**.  
2. Выбрать страницу, которая будет служить входной точкой в справочник (например, “Справочная информация” или “Полезные материалы”).  
3. В зависимости от используемой темы:

   - если тема уже поддерживает SimpleSearch, убедиться, что в шаблоне присутствует поиск;  
   - если нет — добавить на страницу вызов поисковой формы, например, с помощью Twig‑функции:  
     `{{ simplesearch_searchbox() }}`  
     (точный синтаксис берётся из документации к плагину и используемой теме).[web:110]

4. Сохранить изменения страницы.



---

### 6. Проверка работы поиска (план действий)

После успешного запуска Grav (вне Docker или с корректной настройкой контейнера) планируется следующая проверка:

1. Открыть в браузере страницу справочника с установленной формой поиска.  
2. Ввести ключевое слово, которое встречается в одном из справочных материалов (например, название дисциплины, фамилию преподавателя или термин).  
3. Нажать кнопку поиска и убедиться, что отображается список страниц, содержащих данный термин.  
4. Перейти по одной из ссылок и проверить, что страница открывается корректно.

**Скриншот (планируется):**  
– форма поиска на странице;  
– результаты выдачи SimpleSearch с найденными материалами.
